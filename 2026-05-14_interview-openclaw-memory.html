面试官：为什么不直接用 OpenClaw 的 memory，要自己手动辛苦搭？

关于 Athena memory 系统选型的回答策略 · 30 秒版 / 2 分钟版 / 追问应对 / 雷区

⚡30 秒电梯版 · 直接背

ELEVATOR · 5 sentences
"其实我们不是没用 OpenClaw —— PR #1986 就是把 OpenClaw 的 contextPruning 思想对齐过来。但借思想 ≠ 抄代码。
OpenClaw 是 TypeScript 单 agent 设计，Athena 是 Python + 13 agent 共享 workspace + Hatchet durable task。直接 reuse memory 模块要么跨语言 RPC，要么改成 Python —— 等于又造了一遍。
所以选择'借鉴架构，自己实现'，更 fit Athena 的部署形态。"

📋2 分钟完整版 · 主答

① 语言层 语言与生态不匹配
OpenClaw 是 TS 写的，跟 the-platform-lark 一起服务 Slack/IM platform agent。Athena 主代码是 Python（基于 fork 的 nanobot-ai），团队产能在 Python，Bedrock SDK + VectorDB + Hatchet 全部走 Python。
如果直接复用 OpenClaw memory，要么开 sidecar 跨语言通信（增加延迟和故障面），要么改写一遍 —— 改写就等于自建。
② 部署形态 根本不同 · 抽象的根基不同
OpenClaw 设计的是单 agent 独立 workspace，memory 跟 hooks / state 紧耦合。Athena 是13 agent 共享同一份 workspace（Athena / Echo / skin-07 / skin-06 ...）。
OpenClaw 的 GitStore 自动 commit 在 13 process 并发下会 lock 冲突 —— 这不是"抄过来能改"，是抽象根基不同。
③ 借鉴而非全弃 Athena 团队对 OpenClaw 是有学习的
实际上：
PR #1986 align contextPruning progressive disclosure 思想 pgvector 整体方向
但所有这些是把架构思想搬到 Python，不是搬代码本身。
CLOSE 收尾金句
"所以选择是清晰的 trade-off —— 失去现成代码的稳定性，换来fit 自己部署形态的灵活性。
代价是 17+ memory PR、5 次重写，但每次都解决了具体线上问题（13 agent 并发 / Hatchet checkpoint / DeepSeek 成本等），这些问题在 OpenClaw 原始 context 里压根不会出现。"

✨4 个加分点 · 主动抛出去

+ 1
三层 fork 视角
"我们也不是从零写。是 fork 了 nanobot-ai 改造。OpenClaw → nanobot → Athena 是三层关系，Athena 取最适合 Python 业务侧的。"
+ 2
失败案例诚实
"nanobot 框架代码里有 Consolidator / Dream / GitStore，我们试过但发现共享 workspace 跑不起来 → 写了 hooks/compaction.py 替代。这恰恰说明直接 reuse 别人的 memory 抽象在我们场景下行不通。"
+ 3
成本数据
"compaction 5 阶段让 80% 流量走纯字符串（零 LLM 成本），只有 85% 阈值才调 DeepSeek。如果直接用 OpenClaw 的 LLM 摘要，每条消息都烧钱。"
+ 4
Product fit
"Athena 是给团队用的，需要人工 review 关键决策（Auto-Pilot Step 3/4），需要共享知识库做组织记忆。OpenClaw 是个人 chat agent 模式，'agent 自动管所有'反而违反我们的产品哲学。"

❓3 个常见追问 · 应对模板

Q1那为什么不直接 fork OpenClaw 改成 Python？
"fork TS 改 Python 跟从头写没区别。"
fork nanobot-ai 至少保留 Python 生态（Bedrock SDK / loguru / httpx 都直接用），fork OpenClaw 要先把 TS 改 Python 才能开始改业务，工作量翻倍。
Q2自建会不会重复 OpenClaw 已踩过的坑？
"会，确实有。" 所以 PR #1986 那种主动 'align with OpenClaw' 的 PR 就是定期复盘 OpenClaw 学到了什么，我们要不要跟。
这是主动同步思想而不是被动用代码。
Q3如果重来一次，你会选 OpenClaw 还是自建？
"依然自建 —— 因为 Athena 的产品形态（13 agent 共享 + Hatchet durable + IM platform 团队群）跟 OpenClaw 设计场景差距太大。
但我会更早做架构对齐文档（哪些借鉴、哪些不借鉴、为什么），少走一些走弯路的 PR。"
