Athena 架构深扒

从 auto-agent/ 源码出发，看清楚"高频使用的 Athena"实际怎么转
Athena 不是普通聊天 bot —— 它是 TikTok 红人营销活动的 AI 经理。每天定时跑一遍 7 步 autopilot（其中 2 步要在 IM platform 上点按钮人工确认），同时接团队成员的 ad-hoc 提问。底层是 Bedrock Claude + Hatchet durable task + pgvector 长期记忆 三件套。

1一条消息走过的路

用户在 IM 群 @Athena 发消息后，从 webhook 进来到看到回复，要经过 去重 → Hatchet durable task → 5 个内部子步骤 → 卡片更新。Hatchet 这层是关键 —— 它让整个流程容器重启也能续传。

为什么用 Hatchet？ 一次 agent 跑下来可能要几分钟（LLM 调多次 + 工具调用），如果中途容器挂了，普通 background task 就丢了。Hatchet 把任务持久化，重启后能续传 —— Athena 不丢消息的关键。

2难度路由 · 决定调 Sonnet 还是 Opus

Athena 一天处理几千条消息，不能每条都用 Opus（贵 5×）。所以加了 3 层路由：能快判就快判，判不出就调便宜的 LLM 判，最后兜底 regex。

1
硬规则 零成本 · 0ms
纯文本判定 —— 是 greeting？是 image？是 code block？是 stack trace？命中直接分类，不走 LLM。
2
Gemini Nova Micro 判官 $0.000007/call · <1s
硬规则没命中，调最便宜的小模型当裁判，输出 1-5 难度 + 分类标签。便宜到可以为每条消息都跑一次。
3
Heuristic 兜底 regex 评分 · 0ms
Gemini 超时 / 失败时，老 V1 的 regex 评分逻辑兜住，保证绝不挂。
路由结果：
GREETING→难度 1Sonnet
SIMPLE→难度 2Sonnet
CREATIVE→难度 3Sonnet
CODING→难度 4Opus
ANALYSIS→难度 4Opus

5存储分层 · 数据各回各家

Athena 有 5 类数据，分别住在 5 个地方。这是个常见的复杂点 —— 不同生命周期、不同访问模式必须分开。

长期语义记忆
VectorDB · pgvector + Gemini embedding-001 (3072-dim)
存历史对话、用户偏好、过往结论。cosine > 0.9 自动合并去重；> 1 天的条目读出来时自动加 caveat（"这是旧信息"）。
租户 / Agent / 配额
PostgreSQL · src/admin/schema.sql
tenants（租户）· agents（每个 bot）· api_keys（hashed）· usage_events（计费日志）。启动时 ensure_schema() 初始化。
IM platform 卡片生命周期
SQLite · card_lifecycle.db
卡片状态（thinking / working / final）持久化，容器重启不丢。30s 跑一次后台 reconciler 修复孤儿卡片。
Iteration checkpoint
文件系统 · workspace/.checkpoints/
Agent loop 每轮保存一次完整 state。崩了重启可以从最后 checkpoint 续 —— 不用从头重跑。
Workflow 运行状态
FastAPI backend + 本地 JSON fallback
7 步 workflow 的运行记录（哪步在跑、哪步完成）。双重存储：API 不可用时回退本地 JSON，保证不卡。

7已知脆弱点 · 不藏着掖着

CLAUDE.md 里明确列出的 5 个已知坑。改 Athena 代码前最好先扫一遍。

_step_card_cache 内存丢失
每步等人工确认的卡片状态存内存，容器重启就丢，按钮失效 —— 必须重发卡片才行。
_token_cache 单进程
IM token 缓存只在单进程里有效，跑多 worker 时每个 worker 都自己算一份 token，可能被 IM platform 限流。
Workflow 存储双写分裂
FastAPI backend 不可用时回退本地 JSON，但没合并逻辑 —— API 恢复后 DB 和 JSON 状态可能对不上。
Compaction 时机偏晚
在 70% 预算时才触发裁剪，遇到非常长的工具输出（比如 huge grep 结果）单次就能撑爆 —— 还没等到下次裁剪就已经超了。
nanobot-ai pkg 未 pin
PyPI 包没钉版本号，上游静默 API 变更会突然挂掉。重建镜像前最好确认版本。
